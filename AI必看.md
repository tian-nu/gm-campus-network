# AI必看 - 校园网自动认证工具开发指南

## 项目概述
这是一个**校园网自动认证工具 v2.0**，用于广东工贸职业技术学院校园网自动登录。

## 核心功能
1. **CAS认证**：通过 `cas.gzittc.com` 完成单点登录
2. **心跳保持**：定期请求防止掉线
3. **断线重连**：检测断网自动重连
4. **封禁处理**：检测账号封禁状态，自动等待重试

## 关键服务器信息
| 配置项 | 值 |
|--------|-----|
| Portal IP | `10.10.21.129` |
| CAS 域名 | `cas.gzittc.com` |
| Portal 域名 | `xykd.gzittc.edu.cn` |

## 认证流程
1. 访问 `portalScript.do` 获取参数
2. 重定向到 CAS 登录页面
3. 提取表单令牌 `lt`
4. POST 登录数据
5. 跟随重定向到 logout 页面表示成功

## 踩坑记录

### 1. MAC地址获取失败
- **问题**：`uuid.getnode()` 在某些设备返回错误值
- **解决**：使用 `getmac` 命令作为备选方案

### 2. 重定向URL错误
- **问题**：服务器返回 `vl an` 而非 `vlan`
- **解决**：在 `authenticator.py` 第337行手动替换

### 3. 封禁检测
- **问题**：频繁登录导致账号被封禁30分钟
- **解决**：添加封禁关键词检测，自动等待重试

### 4. IP变化未处理（已解决）
- **问题**：切换网络后IP变化，但未重新认证
- **解决**：在 `ReconnectService` 中添加IP变化监测，自动触发重新认证

### 5. 单URL检测误判（已解决）
- **问题**：单个检测URL不可达时误判断线
- **解决**：使用多URL并行检测，多数投票机制

### 6. 频繁重连被封禁（已解决）
- **问题**：固定间隔重连可能导致频繁请求被封禁
- **解决**：实现指数退避策略，连续失败自动延长重试间隔

### 7. 服务异常退出（已解决）
- **问题**：心跳或重连服务异常退出后无人看管
- **解决**：添加看门狗服务，自动监控并重启异常服务

## 新增功能（v2.1）

### 核心优化
1. **多URL并行检测**：3个URL同时检测，多数投票，防误判
2. **IP变化监测**：自动检测IP变化并触发重新认证
3. **指数退避重连**：1s→2s→4s→8s→16s→30s封顶，避免频繁重连
4. **动态检测间隔**：稳定时延长（最长达300秒），断线时缩短（最短10秒）
5. **看门狗保活**：自动监控并重启异常退出的服务

### 用户体验
1. **托盘动态状态**：🟢在线/🔴断开/🟡重连中
2. **静默模式**：仅异常时通知，正常运行不打扰
3. **代理检测**：登录前检测代理，防止被封禁

### 8. 代理导致封禁（已解决）
- **问题**：开着代理登录校园网可能被检测为异常行为，导致封禁30分钟
- **解决**：添加代理检测功能，登录前自动检测并警告用户

## 开发注意事项
1. 所有网络请求需要设置超时时间
2. 日志使用 `logging` 模块，统一格式
3. GUI操作必须在主线程执行（使用 `root.after`）
4. 配置文件使用 UTF-8 编码
5. 后台服务使用 daemon 线程
